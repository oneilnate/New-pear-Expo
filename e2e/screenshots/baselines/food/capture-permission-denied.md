# Capture Screen — baseline notes

These baselines are generated by `pnpm exec playwright test e2e/screenshots` against the
web build. The capture screen requires camera hardware and is not testable headlessly via
Playwright web. Baselines for /food/capture are deferred to EAS Expo Go preview testing
(see AGENTS.md § Expo Go demo).

Baseline state as of F3-E2:
- permission-denied: SafeAreaView, "Camera access required" heading, "Open Settings" green button, "Go back" ghost button
- preview: full-screen Image, "Retake" dark button, "Use this photo" green button in action bar
- uploading: same as preview with ActivityIndicator spinner in place of "Use this photo" text
- error: preview + red error banner above action bar, "Retry" label on primary button
