# Food Player Screen — Playwright Baseline

## Baseline: player.png

The Playwright baseline screenshot (`player.png`) for the `/food/player` route
is generated by CI when the route is first added to `screens.spec.ts`.

The player route (`/food/player`) is not currently included in the Playwright
route manifest (`screens.spec.ts`) because it requires:
1. An authenticated API response to `/api/pods/:id/episode`
2. expo-av Audio.Sound loading state (native, not available in web build)

**Device testing** of the player is done via:
- Expo Go on iPhone (OTA preview per PR)
- Manual API endpoint: `GET https://pear-sandbox.everbetter.com/api/pods/pod_demo_01/episode`
  (live after F4-E3 pipeline lands)

Reference design: IMG_5119 — black background, FoodPod title, 03:03 timer pill,
white bottom sheet with episode title/date/summary and PlayerControls.
